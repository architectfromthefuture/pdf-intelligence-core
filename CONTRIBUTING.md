# Contributing

Thanks for improving this project. Keep changes **scoped** and **truthful**.

## Before you open a PR

1. Run `pytest` from a dev install (`pip install -e ".[dev]"`).
2. Match claims to code—if documentation changes, point to tests or mark behavior as **roadmap**.
3. Do not commit secrets, private datasets, or internal URLs.

## Pull requests

- Prefer small PRs with a clear intent.
- If you add embeddings or benchmarks: pin seeds/configs—or say numbers are illustrative.

## Tone

Professional and direct.
